# GitHub Star管理TUIツール 要件定義・基本設計書 (原文)

## 1. システム概要と要件
目的: 増えすぎたGitHubのスター付きリポジトリを効率的に管理・検索・閲覧するためのターミナル向けアプリケーション。

インターフェース: TUI（Text User Interface）

主な機能:
- GitHub GraphQL APIを用いたスター済みリポジトリの取得とローカルDBへの保存。
- インクリメンタル（差分）同期機能。
- リポジトリへの自動タグ付け機能、およびTUI上からの手動タグ編集（CRUD）機能。
- 手動で編集したタグ履歴を活用し、次回以降の自動タグ付け精度を向上させる「学習機能」。
- すべてのタブ・ペイン（サイドバー、メインリスト等）での一貫したVimライクなスクロール操作（j, k対応）
  - 長い行やカラムに対する水平スクロール操作（h, lキー対応）
  - Tagsタブ（サイドバー）の表示・非表示（トグル）切り替え機能（bキー）
  - リポジトリリストの複数行表示による幅の圧縮対応（DataTableをListViewへリファクタリング）
  - ソート操作時、同じキーを押下した際の昇順・降順（ASC/DESC）トグル機能
- **[Phase 3] ローカルLLM (Ollama) 及びベクトル検索の統合**:
  - Ollama API を用いた高度な自動タグ付け推論 (Few-Shot 対応)
  - 文脈、トピック、説明文に基づく「セマンティック検索」機能 (prefix: `?`)
  - `sqlite-vec` による SQLite 上での高速な類似ベクトル検索
  - アプリケーション内からの LLM モデル自動セットアップ (--setup-llm)

## 2. 技術スタック案
言語: Python 3.12+
TUIフレームワーク: Textual
データベース: SQLite3 (必要に応じて sqlite-vec などの拡張機能)
API通信: httpx (GitHub GraphQL APIアクセス用)
ブラウザ連携: Python標準ライブラリ webbrowser
パッケージ管理: uv
